## 为什么

本地大模型（Ollama）已经普及，但缺少一个轻量、可控的 Plan-Execute Agent 框架来充分利用这些模型的能力。现有的 Agent 框架（如 LangChain）过于厚重，而简单的脚本又缺乏规划、工具调用和错误恢复等核心能力。本项目旨在手搓一个"简单但功能齐全"的 Plan-Execute Agent，以 qwen3.6:35b 为唯一模型后端，实现从任务规划到自主执行的完整闭环。

## 变更内容

- 新增一个 Python 实现的 Plan-Execute Agent，支持通过 Ollama 调用本地大模型
- 新增混合架构调度器：规划阶段由代码控制（确保输出结构化 JSON 计划），执行阶段由模型自主循环（灵活应对意外）
- 新增工具注册表机制：文件读写、Shell 命令执行、代码执行三个基础工具，支持通过注册方式扩展
- 新增三级错误恢复策略：自动重试 → 模型自主修复 → 上报用户
- 新增流式进度输出：实时展示规划、执行、错误恢复各阶段的进度
- 新增自治式交互模式（MVP），后续可扩展审批式和对话式

## 功能 (Capabilities)

### 新增功能
- `plan-engine`: 规划引擎，负责将用户任务拆解为结构化 JSON 计划，包含步骤 ID、动作类型、目标文件和目的描述
- `executor-loop`: 执行循环，负责按计划逐步执行，支持模型自主工具调用和自动循环
- `tool-registry`: 工具注册表，定义工具接口规范，提供文件读写、Shell 执行、代码执行三个内置工具
- `error-recovery`: 错误恢复机制，实现三级降级策略（重试 → 自修复 → 上报）
- `progress-output`: 流式进度展示，实时输出规划、执行、错误各阶段的状态和耗时

## 影响

- **依赖**: Python 3.10+，Ollama 本地服务（需运行 qwen3.6:35b 模型），ollama Python SDK
- **硬件**: 需要约 23GB 可用内存用于加载 qwen3.6:35b 模型
- **代码**: 全新项目，无现有代码受影响
- **安全**: Shell 和代码执行工具需要在受控环境中运行，MVP 阶段默认在用户当前工作目录操作

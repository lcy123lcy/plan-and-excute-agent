
## 新增需求

### 需求:三级错误恢复策略
系统必须实现三级降级错误恢复机制：第一级自动重试，第二级模型自主修复，第三级上报用户。错误必须按级别逐级升级，不可跳过。

#### 场景:第一级自动重试
- **当** 工具执行失败（如文件不存在、命令报错）
- **那么** 系统必须自动重试该工具调用，最多重试 3 次，每次重试之间等待 1 秒

#### 场景:第二级模型自主修复
- **当** 自动重试 3 次后仍然失败
- **那么** 系统必须将错误信息返回给模型，由模型自主决定修复策略（如修改文件路径、调整命令参数），并重新尝试执行

#### 场景:第三级上报用户
- **当** 模型自主修复后仍然失败，或模型无法提出修复方案
- **那么** 系统必须终止当前步骤，输出错误摘要和建议，等待用户手动介入

#### 场景:错误级别不跳过
- **当** 某个步骤发生错误
- **那么** 系统必须严格按照 重试 → 自修复 → 上报 的顺序处理，禁止跳过任何一级

### 需求:错误上下文保留
错误恢复过程中必须保留完整的错误上下文，包括原始错误信息、重试次数、模型修复尝试等，以便上报用户时提供完整信息。

#### 场景:错误链记录
- **当** 一个步骤经历了多级错误恢复
- **那么** 最终输出必须包含完整的错误链：原始错误 → 重试记录 → 模型修复尝试 → 最终结果

### 需求:可恢复与不可恢复错误区分
系统必须区分可恢复错误（如文件不存在、命令语法错误）和不可恢复错误（如权限不足、磁盘满），不可恢复错误必须直接跳到第三级上报用户。

#### 场景:不可恢复错误直接上报
- **当** 工具返回权限不足或磁盘空间不足等不可恢复错误
- **那么** 系统必须跳过第一级和第二级，直接上报用户

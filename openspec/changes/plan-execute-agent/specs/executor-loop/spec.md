## 新增需求

### 需求:按计划逐步执行
执行循环必须按照规划引擎输出的步骤列表依次执行，每完成一步后进入下一步，直到所有步骤完成或遇到不可恢复错误。

#### 场景:顺序执行所有步骤
- **当** 计划包含 4 个步骤且所有步骤均可正常执行
- **那么** 执行循环必须按 id 顺序依次执行每个步骤，最终输出所有步骤的执行结果

#### 场景:步骤执行结果反馈
- **当** 一个步骤执行完成
- **那么** 执行结果必须反馈给模型，作为下一步执行的上下文输入

### 需求:模型自主工具调用
在执行阶段，模型必须能够通过工具调用机制自主决定调用哪个工具、传入什么参数，并处理工具返回的结果。

#### 场景:模型调用文件读取工具
- **当** 执行步骤需要读取文件内容
- **那么** 模型必须通过 tool call 调用 read_file 工具，传入文件路径参数，并基于返回内容决定后续操作

#### 场景:模型调用 Shell 工具
- **当** 执行步骤需要运行命令
- **那么** 模型必须通过 tool call 调用 run_shell 工具，传入命令字符串参数，并基于命令输出决定后续操作

#### 场景:模型自主循环终止
- **当** 模型判断当前步骤已完成
- **那么** 模型必须返回不含 tool call 的文本响应，执行循环据此进入下一个步骤

### 需求:执行阶段模型自主
执行阶段必须由模型自主决定工具调用顺序和次数，代码不干预模型的具体执行逻辑，仅提供工具能力和循环框架。

#### 场景:模型多次调用同一工具
- **当** 一个步骤需要读取多个文件
- **那么** 模型必须能够连续多次调用 read_file 工具，代码不限制调用次数

#### 场景:模型组合调用不同工具
- **当** 一个步骤需要先读文件再写文件
- **那么** 模型必须能够先调用 read_file 再调用 write_file，基于读取结果生成写入内容

## 新增需求

### 需求:流式进度输出
系统必须在规划、执行、错误恢复各阶段实时输出进度信息，用户无需等待整个任务完成即可了解当前状态。

#### 场景:规划阶段进度
- **当** 系统开始规划任务
- **那么** 必须立即输出"规划中..."提示，规划完成后输出计划摘要（步骤数量和目标描述）

#### 场景:执行阶段进度
- **当** 系统开始执行某个步骤
- **那么** 必须输出当前步骤编号、步骤描述和开始时间，执行完成后输出耗时和结果摘要

#### 场景:工具调用进度
- **当** 模型在执行过程中调用工具
- **那么** 必须输出正在调用的工具名称和参数摘要

### 需求:输出格式规范
进度输出必须使用统一的格式规范，包含状态图标、阶段标识和时间信息，确保输出清晰可读。

#### 场景:状态图标使用
- **当** 输出各阶段状态
- **那么** 必须使用以下图标：📋 规划中、✅ 完成、🔧 执行中、⚠️ 警告/重试、 失败

#### 场景:耗时统计
- **当** 一个步骤执行完成
- **那么** 输出必须包含该步骤的耗时（精确到 0.1 秒）

#### 场景:最终汇总
- **当** 所有步骤执行完成
- **那么** 必须输出最终汇总：总步骤数、成功数、失败数、总耗时

### 需求:错误可视化
错误信息必须以醒目的方式展示，包含错误类型、错误详情和建议的修复方向。

#### 场景:错误展示格式
- **当** 执行过程中发生错误
- **那么** 输出必须包含错误级别标识（⚠️ 可恢复 / ❌ 不可恢复）、错误描述和当前恢复策略
